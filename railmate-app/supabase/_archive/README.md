# Archived Schema Files

**Date archived:** 2026-06-19  
**Reason:** Schema conflict resolution — UUID schema is canonical

## What was archived

### `schema.sql`
The original schema used SERIAL (integer) primary keys, a `shohoz_city` column
coupling every station to the Shohoz/Rail Sheba API, and `off_days` (a text[]
of day names) instead of `days_of_week` (a SMALLINT[] of day numbers, Sunday=0).

This schema was **incompatible** with:
- The Master Reference Document Part 07 (which specifies UUID PKs and `days_of_week`)
- The mobile app's `supabase.rpc('search_trains', ...)` call (Part 08 §8.2)
- Legal compliance (Part 02 §2.1): `shohoz_city` baked in a dependency on
  the Shohoz API, which is explicitly forbidden without a written partnership

### `seed_original.sql`
The seed file that accompanied the old schema. Kept for historical reference
only — the data content was already migrated to the UUID schema in the
`seed.sql` one level up (generated by `scripts/generate_canonical_seed.mjs`).

## What replaced them

The canonical schema is **`migrations/001_initial_schema.sql`** (UUID PKs,
`days_of_week`, `search_trains()` RPC) plus **`migrations/002_community_delay_schema.sql`**
(community_reports, report_votes, public_delay_aggregates view).

The active seed data is **`seed.sql`** (one level up from this directory),
which is keyed to the UUID schema via `uuid_generate_v5`.

## Do not restore these files

If you need to understand what data was here, read `seed.sql` — it contains
all 52 stations, 133 trains, and the 43 verified stop rows, all in the
canonical UUID format.
